前言

在完成了 Metasploitable2 中 vsFTPd 2.3.4 后门漏洞 的学习后，我继续对靶机开放的服务进行分析。这次学习的目标是 23 端口（Telnet）。

刚开始学习时，我有一个误区：我认为每一个开放的端口都应该对应一个漏洞，就像 vsFTPd 一样可以直接利用。然而，通过这次学习，我认识到，并不是所有服务都存在漏洞，很多时候攻击成功的原因并不是软件漏洞，而是配置不当或弱口令。

一、端口发现

首先使用 Nmap 对靶机进行服务扫描：

nmap -sV 192.168.93.148

扫描结果显示：

23/tcp open  telnet

说明目标主机开放了 Telnet 服务。

二、什么是 Telnet？

Telnet 是一种经典的远程登录协议，默认使用 TCP 23 端口。

它的作用非常简单：

允许用户通过网络远程登录另一台计算机，并获得该主机的命令行终端。

例如：

我的电脑
      │
      │
   Telnet
      │
      ▼
Linux服务器

登录成功以后，就相当于坐在服务器前操作一样，可以执行 Linux 命令。

三、Telnet 为什么不安全？

Telnet 最大的问题就是：

所有通信内容都是明文传输。

包括：

  • 用户名
  • 密码
  • 输入的命令
  • 返回结果

如果攻击者能够监听网络流量，就有可能直接获取用户名和密码。

因此，目前生产环境几乎都已经使用 SSH 来代替 Telnet。

两者的主要区别如下：

TelnetSSH
明文传输加密传输
默认23端口默认22端口
安全性较低安全性较高
目前很少用于生产环境广泛使用

四、使用 Telnet 连接目标主机

在 Kali Linux 中，可以直接使用 Telnet 客户端连接目标主机：

telnet 192.168.93.148

连接成功后出现：

Trying 192.168.93.148...
Connected to 192.168.93.148.
Escape character is '^]'.

metasploitable login:

这里说明：

  • 网络连接正常；
  • TCP 三次握手已经完成；
  • 成功连接到目标主机的 Telnet 服务；
  • 系统正在等待输入用户名。

五、使用默认账号登录

Metasploitable2 为了教学方便，保留了默认账户：

用户名：

msfadmin

密码：

msfadmin

输入后成功登录：

Last login: Tue Jul 7 02:30:11 EDT 2026 on tty1

Linux metasploitable ...

msfadmin@metasploitable:~$

此时说明：

已经成功获得目标主机的远程 Shell。

这里需要特别注意：

这并不是利用了 Telnet 漏洞。

真正成功的原因是：

系统使用了默认账号和默认密码。

因此，这属于弱口令攻击，而不是漏洞利用。

六、获得 Shell 后的信息收集

登录成功以后，并不会立刻进行提权，而是首先确认当前权限和系统信息。

1、查看当前用户

whoami

输出：

msfadmin

说明当前登录用户为 msfadmin。

2、查看用户权限

id

输出：

uid=1000(msfadmin)
gid=1000(msfadmin)

其中：

  • UID=0 表示 root；
  • UID=1000 表示普通用户。

因此，目前获得的是普通用户权限，而不是最高权限。

此外，还可以看到当前用户所属的多个用户组，例如：

  • adm
  • audio
  • video
  • lpadmin
  • sambashare

这些组信息在后续 Linux 提权过程中可能具有重要参考价值。

3、查看当前目录

pwd

输出：

/home/msfadmin

说明当前位于用户主目录。

Linux 中常见的重要目录包括：

目录作用
/根目录
/home普通用户目录
/rootroot 用户目录
/etc系统配置文件
/var日志和数据库
/tmp临时文件
/usr应用程序

4、查看网络连接

netstat -ant

可以看到大量监听端口，例如：

21   FTP
23   Telnet
80   HTTP
139  Samba
445  Samba
3306 MySQL
5432 PostgreSQL
8180 Tomcat

这些端口与之前使用 Nmap 从外部扫描得到的结果基本一致。

说明：

  • Nmap 是站在攻击者角度观察目标开放了哪些服务；
  • netstat 是站在目标主机内部查看自己正在监听哪些端口。

两者分别对应外部视角和内部视角，对渗透测试都非常重要。

另外，在输出中还能看到：

192.168.93.148:23
192.168.93.146:53132
ESTABLISHED

其中：

  • 192.168.93.148 为靶机；
  • 192.168.93.146 为 Kali 主机。

说明当前 Telnet 会话已经成功建立，并保持连接状态。

七、本次学习总结

通过本次实验，我不仅学会了如何使用 Telnet 登录远程 Linux 主机，更重要的是理解了一个渗透测试中的重要概念：

开放服务并不意味着一定存在可利用漏洞。

不同服务需要采用不同的攻击思路：

  • 有些服务存在软件漏洞，可以直接利用漏洞获得权限，例如 vsFTPd 后门漏洞；
  • 有些服务本身没有漏洞，但由于管理员使用了默认账号或弱密码，同样可能导致服务器被攻陷，例如本次实验中的 Telnet；
  • 还有一些服务主要用于信息收集，需要结合其他漏洞进行进一步攻击。

因此，在实际渗透过程中，应先识别目标服务，再分析其攻击面，选择合适的方法，而不是盲目寻找漏洞。

本次实验还掌握了以下基础命令：

命令作用
telnet IP连接 Telnet 服务
whoami查看当前用户
id查看用户 UID 和所属组
pwd查看当前工作目录
netstat -ant查看网络连接和监听端口

这些命令都是后续 Linux 信息收集和权限分析的重要基础。

学习心得

这次学习让我意识到，渗透测试不仅仅是寻找漏洞和运行 Exploit，更重要的是理解目标服务的工作原理，并根据实际情况选择合适的攻击方式。Telnet 本身并没有被利用，而是由于默认账号和弱口令导致了远程登录成功。这也让我认识到，安全风险不仅来源于程序漏洞，错误的配置和不安全的账户管理同样可能成为攻击入口。

后续我将继续学习 Metasploitable2 中的其他服务，如 Samba、Tomcat、MySQL 等，进一步掌握不同类型服务的渗透思路，逐步建立完整的网络渗透测试知识体系。
