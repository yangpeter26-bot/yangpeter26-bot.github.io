# 卷积神经网络（CNN）

## 是什么

专门用来处理图片的神经网络。

## 为什么不用全连接层

```
一张图片100万个像素
全连接层：每个神经元看全部100万像素 → 参数太多，训练不动
CNN：    每个神经元只看一小块（3×3）→ 参数很少，能训练
```

CNN利用图片的两个特性：
- 平移不变性：猪在左上角还是右下角，都是猪
- 局部性：看局部就够了，不用看全图

## CNN的三个核心组件

### 1. 卷积层（提取特征）

用一个小方块（卷积核，比如3×3）在图片上滑动，每次看一小块区域，提取边缘、纹理等特征。

卷积核 = 可学习的参数，通过训练学出来。

### 2. 池化层（压缩信息）

把图片变小，保留重要信息。

最大池化：在2×2方块里只留最大值。
28×28 → 池化后 → 14×14

池化没有参数，是固定操作。

### 3. 全连接层（做决定）

卷积和池化做完后，综合所有特征判断这是什么。

## LeNet结构（第一个CNN模型）

```
图片 → 卷积 → 池化 → 卷积 → 池化 → 全连接 → 全连接 → 答案
       提取特征  压缩   提取特征  压缩    判断      判断
```

具体：
```
输入(1×28×28)
  → 卷积(5×5, 6个通道) + Sigmoid + 池化(2×2)
  → 卷积(5×5, 16个通道) + Sigmoid + 池化(2×2)
  → 展平
  → 全连接(120) → 全连接(84) → 全连接(10)
```

通道数变化：1 → 6 → 16
尺寸变化：28×28 → 14×14 → 5×5

## 关键概念

| 概念 | 说明 |
|------|------|
| 卷积核 | 小方块（3×3），在图片上滑动提取特征 |
| 填充(Padding) | 在图片边缘填0，防止图片越来越小 |
| 步幅(Stride) | 卷积核每次跳着走，让图片快速变小 |
| 通道 | 多个卷积核 → 多个输出 → 提取多种特征 |
| 感受野 | 网络越深，每个神经元看到的区域越大 |

## 一句话总结

CNN = 用小放大镜看局部（卷积）+ 压缩信息（池化）+ 最终判断（全连接）

## 卷积核核心问题汇总

### 问题 1：卷积核是怎么得来的？

**卷积核不是人工设计的，而是通过反向传播和梯度下降，从训练数据中"学习"出来的。**

- 初始状态：随机小数（如 Xavier/Kaiming 初始化）
- 训练过程：前向传播 → 计算损失 → 反向传播求梯度 → 更新权重
- 最终状态：参数收敛，固化成能识别特定特征的滤波器

> 比喻：卷积核像一块"橡皮泥"，初始是随机形状，训练数据是"模具"，梯度下降是"按压的手"，最终被压成能匹配特定特征的形状。

### 问题 2：卷积核的参数是固定的吗？

**分阶段看——训练时变化，推理时固定。**

| 阶段 | 参数状态 | 说明 |
|------|---------|------|
| 训练阶段 | 动态变化 | 每批数据都通过梯度下降更新参数 |
| 推理阶段 | 完全固定 | 参数冻结，用同一套模板处理所有输入 |

> 比喻：像流水线上的模具——制作过程需要反复打磨（训练），一旦做好就固定不变（推理）。

### 问题 3：一个卷积核为什么只能检测一种特征？

**卷积核的本质是一个"固定的模板匹配器"。**

- 数学本质：**点积运算 = 相似度度量**
- 卷积核 = 高维空间（如 3×3=9 维）中的**一个固定方向向量**
- 输入局部窗口 = 同一个空间中的另一个向量
- 只有当两个向量方向一致时，点积结果才大（激活强）；方向偏离越远，响应越小

> 一个核只有一个"方向"，所以只能匹配一种特征。

### 问题 4：具体是怎么实现"只认一种"的？（数值示例）

用一个检测**垂直边缘**的 3×3 卷积核：

```
-1   0   1
-1   0   1
-1   0   1
```

在图片上滑动时：

| 输入局部图案 | 点积结果 | 结论 |
|-------------|---------|------|
| 左亮右暗（匹配） | 绝对值很大 | 检测到竖边 |
| 均匀灰色（无关） | 约等于 0 | 无反应 |
| 右亮左暗（相反） | 绝对值很大但符号相反 | 检测到（取绝对值或 ReLU 后） |

### 问题 5：训练之前，卷积核知道自己的分工吗？

**完全不知道！训练前所有核都是"废物"，随机且毫无分工。**

那为什么训练后会形成分工？

| 原因 | 说明 |
|------|------|
| 随机初始化的蝴蝶效应 | 不同核初始值不同 → 梯度不同 → 被推向不同方向 |
| 数据多样性逼迫分工 | 若所有核都检测同一种特征，其他特征无人响应 → Loss 高 → 被迫分化 |
| 局部最优 | 不同核掉进损失函数空间里的不同"坑"，各自安家 |

> 比喻：不是提前规划了"你负责竖线"，而是"随机出生 + 后天环境（数据）+ 生存压力（Loss）"共同逼迫它们**自发分化**。

### 问题 6：一个卷积层怎么检测多种特征？

**靠"多通道"堆叠——每个卷积层不只有一个核，而是有几十上百个。**

| 卷积层 | 卷积核数量 | 每个核检测的特征 |
|--------|-----------|-----------------|
| 浅层 | 64 个 | 64 种边缘/颜色/纹理方向 |
| 中层 | 256 个 | 256 种复杂纹理组合 |
| 深层 | 512 个 | 512 种高级语义部件 |

> 比喻：一个卷积层就像一支有 64 个不同"专家"的团队，每个专家只负责看一种特征，合在一起就能读懂完整画面。

### 一句话终极总结

> **卷积核 = 随机出生的"白纸"，在训练中被梯度下降和 Loss 压力逼成了各自分工的"固定模板"，一个核只有一个方向，所以只能检测一种特征；一个层有几十上百个核，所以能覆盖多种特征。**
